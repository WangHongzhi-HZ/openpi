================================================================================
四、数据流 (Pi0.5 + ForceVLA)
================================================================================

  训练 (compute_loss):
    1. embed_prefix: 图像(SigLIP) → tokens + 离散 prompt (含 prio state)
    2. embed_suffix: 噪声动作 + time → adaRMS_cond
       + state[:, 7:13] → (上游已 forward fill) → nan_to_num 兜底 → force_in_proj → force_tokens
    3. LLM: [prefix, suffix], adarms → (prefix_out, suffix_out)
    4. LIMoE: concat([prefix_out, force_tokens]) → MoE → limoe_out
    5. 输出: action_out_proj(limoe_out[:, -horizon:] + suffix_out[:, -horizon:])
    6. Loss: MSE(v_t - u_t), u_t = noise - actions

  推理 (sample_actions):
    1. prefix → prefix_out_fix + kv_cache
    2. while_loop (t=1→0):
       a. suffix + force_tokens
       b. LLM with kv_cache → suffix_out
       c. LIMoE(prefix_out_fix, force_tokens) → limoe_out
       d. v_t = action_out_proj(limoe_out + suffix_out)
       e. x_{t+dt} = x_t + dt * v_t
    3. 返回 x_0


================================================================================
五、已修复的问题 (5)
================================================================================

问题 1: Force 切片位置错误 (state[:, -force_dim:] → padding 零)
  修复: 新增 force_start_index 字段，使用显式索引 state[:, 7:13]

问题 2: 原版 ForceVLA 推理 bug (pi0_force.py:338 使用 prefix_out=None 拼接)
  修复: 使用外层捕获的 prefix_out_fix

问题 3: 原始 NaN force 值 → norm_stats null → 训练崩溃 (全链路)
  修复 (2026-06-23 更新: 改为 forward fill，避免传感器瞬断跳变到 0):
    a. convert_custom2lerobot_single_eef.py: pd.DataFrame.ffill 沿时间轴向前填充 (源头)
    b. compute_norm_stats.py: pd.DataFrame.ffill (统计，残存 NaN 兜底)
    c. pi0.py embed_suffix: jnp.nan_to_num (训练/推理 — 单帧无时间轴，仅兜底)
    d. episodes_stats.jsonl: NaN→0.0 (LeRobot 元数据修复)

问题 4: flaxformer/dense.py 硬依赖 aqt (Google量化库) → import 链断裂
  修复: from __future__ import annotations + aqt 懒加载 (use_aqt=False 时无需)

问题 5: CheckpointWeightLoader 拒绝含新层的模型结构
  修复: 新增 extra_missing_regex 字段，标记 force_in_proj/limoe 为随机初始化

问题 6 (已知，不处理): TokenizePrompt 将 force 也 tokenize 到离散 prompt
  LIMoE 信号强度 >> 离散 token，不影响收敛。

问题 7 (已知): 数据集中 force dim 2 (fz) 含极端值 -40.57
  quantile normalization 自动压到 [q01, q99] 区间，不影响训练。
